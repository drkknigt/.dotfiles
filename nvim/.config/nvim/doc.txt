                                            NEOVIM EDITOR | MY CONFIG | AND THE REST
                                            ========================================


Neovim is a refactor of vim with same philosophy as vim and with extra features and kind of like extension of vim.
Neovim is vim with good parts and more. Neovim is a Vim-based text editor engineered for extensibility and usability.
It has tons of great plugins and it will soon be a ide

VERSION :neovim 0.7.0 

# features and customizations
-----------------------------

* Plugin management with vim plug
* match chars and programming syntax with %
* fast snippet engine with luasnip
* Debugger protocol and ui to debug the programming language code
* tons of themes
* fuzzy finders for lsp buffers files patterns and much more with telescope and fzf-lua
* lsp, buffer, path , command line and other completion and lsp signature for functions
* vim vim-sandwich for effectively surrounding text with chars and tags 
* make dirs and files effeciently with mkdir.nvim
* dev icons and attractive fonts
* treesitter for syntax parsing and higlighting and other programming tasks like programming text object and refactoring
* hop anywhere in buffer with easy motions using hop.nvim
* impressive and very configurable statusline with lualine 
* tab bar like vscode with bufferline
* file tree explorere with nvim tree
* autocomplete tags with auto tag and colors for brackets with nvim-ts-rainbow
* auto insertions for brackets and other pairs using nvim-autopairs 
* watch your keybindings in realtime with which-key.nvim and easily see your bindings
* easy terminals with toggleterm.nvim
* comment any programming language with comment.nvim
* easily install lsp for programming language with nvim-lsp-installer 
* vscode-like pictorgrams with lspkind-nvim 
* colorizer to get hex colors with colorizer.nvim
* check tags and symbols in the file with symbols-outline 
* very attractive indent line with indent-blankline.nvim
* easy formatting and linting with null-ls 
* preview code,functions , classes with goto-preview and treesitter
* gitsigns for git
* run code with jaq-nvim
* project managament with telescope-proejct.nvim

#VIM RC / INIT.VIM

NOTE:    'A file containing initialization commands is generically called
        a "vimrc" or config file.  It can be either Vimscript ("init.vim") or
        Lua ("init.lua"), but not both.'

1.) set nocompatible:-> disables default vim bindings

2.) set nu :- > set number lines

3.) set rnu:-> sets relatvive number line

4.) set clipboard=unnamed :-> When included, Vim will use the clipboard register '*'
			for all yank, delete, change and put operations which
			would normally go to the unnamed register.  When a
			register is explicitly specified, it will always be
			used regaless of whether "unnamed" is in 'clipboard'
			or not.  The clipboard register can always be
			explicitly accessed using the "* notation.

5.) set clipboard=namedplus :->  A variant of the "unnamed" flag which uses the
			clipboa register '+' (|quoteplus|) instead of
			register '*' for all yank, delete, change and put
			operations which would normally go to the unnamed
			register.  When "unnamed" is also included to the
			option, yank and delete operations (but not put)
			will additionally copy the text into register
			'*'. See |clipboa|.

61.) function CleanEmptyBuffers()
    let buffers = filter(range(1, bufnr('$')), 'buflisted(v:val) && empty(bufname(v:val)) && bufwinnr(v:val)<0 && !getbufvar(v:val, "&mod")')
    if !empty(buffers)
        exe 'bw ' . join(buffers, ' ')
    endif
endfunction :-> clear all the empty buffers
62.) :command Be :call CleanEmptyBuffers() :-> custom command to call the empty buffer fuction
63.) nnoremap <silent> <leader>be :call CleanEmptyBuffers()<CR> :-> map the empty buffer command to the eys
